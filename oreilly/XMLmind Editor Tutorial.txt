    3.5.2:</p>

<div class="titlepage"><div><div><h2 class="title">Using XXE</h2></div></div></div>
<p id="a_program_like_">A program like XXE puts a word-processing face on XML, making it
      more user-friendly while checking the markup validity. The Element Bar
      (see <a class="xref" href="ch01.html#elementbar" title="Figure 1-1. The Element Bar shows the XML hierarchy">Figure 1-1</a>), or Node Path, at the top of your
      screen lets you know where you are in the XML structure at any given
      point (whether you are in a section, title, paragraph, list, etc.).
      Clicking on an element in the Element Bar will display a red box around
      the element in the document, making it easy to identify where it starts
      and ends.</p>

<div class="sect2" id="splitting_and_merging_sections">
<div class="titlepage"><div><div><h3 class="title">Splitting and Merging Sections</h3></div></div></div>
<p id="if_you_need_to__id1">If you need to split up a section, select the paragraph that you
        want to start the new section by selecting the <code class="literal">para</code> in the Element Bar, and then choose
        DocBook→Promote to create a new section with that paragraph.</p>
<p id="merging_two_sec">Merging two sections can be a bit more difficult. If you are
        just moving a few paragraphs into another section, you can cut and
        paste the text from one section to the other and delete the second
        section title. If you have a more complex section to move, use the
        steps below:</p>
<div class="orderedlist" id="make_sure_the_t_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="make_sure_the_t_id2">Make sure the two sections you want to join are adjacent to
            each other, with the section heading that you want to keep
            positioned first.</p></li>
<li class="listitem"><p id="select_the_firs">Select the first section in the Element Bar.</p></li>
<li class="listitem"><p id="selectselect_a">Select→Select All Children (display boxes will surround all
            the individual elements that make up that section).</p></li>
<li class="listitem"><p id="choose_editcop">Choose Edit→Copy (Cut isn’t an option for a complex
            section).</p></li>
<li class="listitem"><p id="select_the_titl">Select the <code class="literal">title</code> element
            of the second section you want to join.</p></li>
<li class="listitem"><p id="choose_editpas">Choose Edit→Paste and paste the first section into the
            <code class="literal">title</code> element of the second
            section. This will replace the second section’s title with the
            first section’s title, and insert all the other elements of the
            first section at the beginning of the second section.</p></li>
<li class="listitem"><p id="now_go_back_and">Now go back and delete the first section.</p></li>
</ol></div>
</div>
<div class="sect2" id="using_elements_correctly">
<div class="titlepage"><div><div><h3 class="title">Using Elements Correctly</h3></div></div></div>
<p id="the_element_bar">The Element Bar at the top of XXE will show you which element
        you have selected. <span class="emphasis"><em>Block elements</em></span> are
        paragraph-level elements (e.g., <code class="literal">programlisting</code>). <span class="emphasis"><em>Inline
        elements</em></span> go within block elements (e.g., <code class="literal">emphasis</code>). The Element Bar’s default
        description of text in any element is <code class="literal">#text</code>. You can select <code class="literal">#text</code> and convert to an element that better
        suits your needs, like <code class="literal">literal</code>.</p>
<p id="for_example_if">For example: if you want to emphasize a word or phrase by
        putting it in italics, highlight the text and click the Convert button
        in the Edit panel. In the field that appears, start typing the name of
        the inline element you wish to apply, in this case <code class="literal">emphasis</code>.</p>
<p id="the_convert_but">The Convert button is shown in <a class="xref" href="ch01.html#keyCapWrap" title="Figure 1-13. Using the Convert button">Figure 1-13</a>.
        You can also use the formatting buttons in the toolbar (above the
        Element Bar) to convert text to emphasis, links, plain text, etc. This
        is often faster than using the Convert button for these basic
        elements.</p>
<div class="figure-float"><div class="figure" id="keyCapWrap">
<div class="figure-contents"><div class="mediaobject"><img style="width: 378; " src="http://orm-chimera-prod.s3.amazonaws.com/1234000001577/images/keyCapWrap.png" alt="Using the Convert button"></div></div>
<div class="figure-title">Figure 1-13. Using the Convert button</div>
</div></div>
<p id="to_insert_a_bul">To insert a bulleted list, choose “Add itemizedlist” from the
        toolbar to insert the first bullet point. For a numbered list, choose
        “Add orderedlist.” For a term-definition list, choose “Add
        variablelist.”</p>
<p id="if_you_dont_se">If you don’t see the element you want as an option for your
        highlighted text, it probably means it isn’t a valid option. Try
        placing your cursor at a different level in the hierarchy. Only the
        elements that are valid for your location in the hierarchy according
        to the DTD are displayed as options. (This is a very useful feature of
        XXE as well as a potentially aggravating one, if you can’t figure out
        why the program is not letting you do something.)</p>
<p id="similarly_if_y">Similarly, if you’ve cut text from one spot and are having
        trouble pasting it, make sure you are pasting into the same element.
        XXE won’t let you paste somewhere invalid. Try moving up or down the
        hierarchy, or use the Paste, Paste Before..., or Paste After...
        buttons in the toolbar.</p>
</div>
<div class="sect2" id="inserting_tables">
<div class="titlepage"><div><div><h3 class="title">Inserting Tables</h3></div></div></div>
<p id="there_are_two_o">There are two options for tables: formal and informal. A formal
        table will have a number and title, making it better suited for
        cross-referencing. An informal table lacks a number and title and is
        more suited for a small table interspersed with normal text that won’t
        need to be referenced later. The easiest way to add a table to a
        document is to choose the “Add table” button in the toolbar (shown in
        <a class="xref" href="ch01.html#tableInsert" title="Figure 1-14. Inserting a table">Figure 1-14</a>). The most common type of table you
        will probably use is <code class="filename">table(head_row)</code>—that is, a
        formal table with a row at the top serving as the header.</p>
<div class="figure-float"><div class="figure" id="tableInsert">
<div class="figure-contents"><div class="mediaobject"><img style="width: 405; " src="http://orm-chimera-prod.s3.amazonaws.com/1234000001577/images/tableInsert.png" alt="Inserting a table"></div></div>
<div class="figure-title">Figure 1-14. Inserting a table</div>
</div></div>
<p id="if_you_need_to__id2">If you need to add rows or columns to your table (the default is
        three rows by two columns), go to DocBook→Column→Insert Before, or
        After or DocBook→Row→Insert Before or After.</p>
<p id="for_a_formal_ta">For a formal table, make sure you select the <code class="literal">table</code> element and give it an <code class="literal">id</code> attribute, as illustrated in <a class="xref" href="ch01.html#tableID" title="Figure 1-15. Applying an id attribute to a table">Figure 1-15</a>. This is a unique identifier that will be used
        when you add a cross-reference to it.</p>

<div class="sect2" id="figures_in_xxe">
<div class="titlepage"><div><div><h3 class="title">Inserting Figures</h3></div></div></div>
<p id="figures_are_sim">Figures are similar to tables in that they can be either formal

<div class="titlepage"><div><div><h3 class="title">Inserting Hyperlinks</h3></div></div></div>
<p id="as_explained_in">As explained in <a class="ulink" href="http://chimera.labs.oreilly.com/books/1234000000058/ch02.html#inserting_hyperlinks" target="_top">“Hyperlinks”</a>

        create: those with character data and those without. Here’s the markup
        for each:</p>
<div class="variablelist" id="with_character_"><dl class="variablelist">
<dt><span class="term">With character data</span></dt>
<dd><p id="ulink_urlhtt_id1"><code class="literal">&lt;ulink
              url="http://oreilly.com"&gt;O'Reilly&lt;/ulink&gt;</code></p></dd>
<dt><span class="term">Without character data</span></dt>
<dd><p id="ulink_urlhtt_id2"><code class="literal">&lt;ulink
              url="http://www.oreilly.com"/&gt;</code></p></dd>
</dl></div>
<p id="to_add_a_hyperl_id1">To add a hyperlink with character data in XXE, type the
        character data as regular text (e.g., O’Reilly). Select the text,
        click the “Convert to link” button in the top toolbar, and choose
        <code class="literal">ulink</code> as shown in <a class="xref" href="ch01.html#ulink" title="Figure 1-21. Convert to ulink">Figure 1-21</a>.<a href="#ftn.id507480" class="footnote"><sup class="footnote" id="id507480">[1]</sup></a>If you have the ORM customizations installed, you will

        &lt;ulink&gt;! Needs url&gt;&gt;</em></span>. Go to the Attributes
        panel and enter the <code class="literal">url</code> there (see
        <a class="xref" href="ch01.html#urlAttribute" title="Figure 1-22. Entering the URL">Figure 1-22</a>). It must be a full URL starting with
        <span class="emphasis"><em>http://</em></span>, not <span class="emphasis"><em>www</em></span>. This will
        produce a clickable link in the PDF and electronic versions of the
        book.</p>
<p id="to_add_a_hyperl_id2">To add a hyperlink without character data in XXE, simply insert
        a <code class="literal">ulink</code> element via the Edit panel, and fill in the
        <code class="literal">url</code> attribute as described above.</p>

<div class="sect2" id="working_with_index_terms">
<div class="titlepage"><div><div><h3 class="title">Working with Index Terms</h3></div></div></div>
<p id="the_index_is_au">The index is automatically generated from
        <code class="literal">indexterm</code>s in the component book files. See
        the <a class="ulink" href="http://atlas.labs.oreilly.com/books/1234000001578/" target="_top">O'Reilly
        Indexing Guidelines</a> for more information on indexing
        in DocBook. The following instructions assume the ORM customizations
        are installed.</p>
<div class="warning" id="if_you_are_an_a_id1"><p id="if_you_are_an_a_id2">If you are an author writing a first edition manuscript, you
          do <span class="emphasis"><em>not</em></span> need to worry about indexterms; indexing
          will be done by a professional indexer once your book is in
          production.</p></div>
<div class="sect3" id="inserting_a_new_index_entry">
<div class="titlepage"><div><div><h4 class="title">Inserting a new index entry</h4></div></div></div>
<p id="to_insert_a_new">To insert a new entry:</p>
<div class="orderedlist" id="place_the_curso_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="place_the_curso_id2">Place the cursor where you want the entry, and hit ESC-i.
              This will open a pop-up window that lets you type in primary,
              secondary, and tertiary index terms, separated by
              slashes.</p></li>
<li class="listitem"><p id="hit_enter_and_t">Hit Enter and the index entry will appear in gray.</p></li>
</ol></div>
<p id="to_insert_a_se">To insert a “See” or “See also” entry:</p>
<div class="orderedlist" id="after_creating__id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="after_creating__id2">After creating the entry as outlined above, select the
              <code class="literal">primary</code> element (or
              <code class="literal">secondary</code> or <code class="literal">tertiary</code>,
              depending on what the “See”/“See also” entry refers to) in the
              Element Bar to highlight it.</p></li>
<li class="listitem"><p id="select_insert_">Select “Insert After” (or hit Cmd-j) to insert the
              <code class="literal">see</code> or <code class="literal">seealso</code>
              element.</p></li>
<li class="listitem"><p id="enter_the_appro">Enter the appropriate text.</p></li>
</ol></div>
<p id="to_insert_an_in">To insert an index range:</p>
<div class="orderedlist" id="start_by_insert_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="start_by_insert_id2">Start by inserting a new entry as outlined above.</p></li>
<li class="listitem"><p id="select_the_inde">Select the <code class="literal">indexterm</code> in the Element Bar
              to highlight it. You should see a red box surrounding it.</p></li>
<li class="listitem"><p id="in_the_attribut_id1">In the Attributes panel, go to the “class” row and choose
              <code class="literal">startofrange</code>.</p></li>
<li class="listitem"><p id="with_indexterm_">With <code class="literal">indexterm</code> still highlighted, go to
              the “id” row and enter a name. The name can be anything you
              want, but we suggest you follow a format. For example, if you’re
              working in Chapter 2 and the term is <code class="literal">foo</code>,
              then the <code class="literal">id</code> might be
              <code class="literal">ch02_foo</code>.</p></li>
<li class="listitem"><p id="go_to_the_place">Go to the place in the text where the range should end,
              and insert another <code class="literal">indexterm</code> using ESC-i. But
              don’t actually enter a term in the pop-up window; just press
              “OK” to create an empty <code class="literal">indexterm</code>.</p></li>
<li class="listitem"><p id="select_the_empt">Select the empty <code class="literal">indexterm</code>.</p></li>
<li class="listitem"><p id="in_the_attribut_id2">In the Attributes panel, go to the “class” row and choose
              <code class="literal">endofrange</code>.</p></li>
<li class="listitem">
<p id="with_the_empty_">With the empty <code class="literal">indexterm</code> still
              highlighted, go to the “startref” row and enter the exact same
              <code class="literal">id</code> you used in step 4. If you don’t use the
              same text in the same case, you will get a validity error that
              looks something like this:</p>
<pre class="screen" id="reference_to_no">reference to non-existent ID "ch02_fot" [cvc-id.1]</pre>
</li>
</ol></div>
</div>
<div class="sect3" id="editing_existing_entries">
<div class="titlepage"><div><div><h4 class="title">Editing existing entries</h4></div></div></div>
<p id="to_search_for_i">To search for index entries using XXE:</p>
<div class="orderedlist" id="open_the_bookx_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="open_the_bookx_id2">Open the <code class="filename">book.xml</code> file.</p></li>
<li class="listitem"><p id="with_the_cursor">With the cursor at the top of the document, hit ESC-G to
              open the Find Element pop up.</p></li>
<li class="listitem"><p id="in_the_element">In the “Element” field, type in
              <code class="literal">indexterm</code>. If you want to find a particular
              term, type it in the “Containing Text” field.</p></li>
<li class="listitem"><p id="hit_enter_to_ju">Hit Enter to jump to the first instance in the book. Hit
              Cmd-A to jump to the next instance.</p></li>
<li class="listitem"><p id="once_you_find_t">Once you find the entry you want, you won’t be able to
              edit it because the data is contained in the chapter file, not
              the <code class="filename">book.xml</code> file that you’re searching in.
              So hit Cmd-Shift-E to open the referenced file and jump to the
              entry.</p></li>
<li class="listitem"><p id="edit_and_save">Edit and save.</p></li>
<li class="listitem"><p id="go_back_to_book">Go back to <code class="filename">book.xml</code> to do your next
              search.</p></li>
</ol></div>
</div>

<div class="sect2" id="splitting_and_merging_sections">
<div class="titlepage"><div><div><h3 class="title">Splitting and Merging Sections</h3></div></div></div>
<p id="if_you_need_to__id1">If you need to split up a section, select the paragraph that you
        want to start the new section by selecting the <code class="literal">para</code> in the Element Bar, and then choose
        DocBook→Promote to create a new section with that paragraph.</p>
<p id="merging_two_sec">Merging two sections can be a bit more difficult. If you are
        just moving a few paragraphs into another section, you can cut and
        paste the text from one section to the other and delete the second
        section title. If you have a more complex section to move, use the
        steps below:</p>
<div class="orderedlist" id="make_sure_the_t_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="make_sure_the_t_id2">Make sure the two sections you want to join are adjacent to
            each other, with the section heading that you want to keep
            positioned first.</p></li>
<li class="listitem"><p id="select_the_firs">Select the first section in the Element Bar.</p></li>
<li class="listitem"><p id="selectselect_a">Select→Select All Children (display boxes will surround all
            the individual elements that make up that section).</p></li>
<li class="listitem"><p id="choose_editcop">Choose Edit→Copy (Cut isn’t an option for a complex
            section).</p></li>
<li class="listitem"><p id="select_the_titl">Select the <code class="literal">title</code> element
            of the second section you want to join.</p></li>
<li class="listitem"><p id="choose_editpas">Choose Edit→Paste and paste the first section into the
            <code class="literal">title</code> element of the second
            section. This will replace the second section’s title with the
            first section’s title, and insert all the other elements of the
            first section at the beginning of the second section.</p></li>
<li class="listitem"><p id="now_go_back_and">Now go back and delete the first section.</p></li>
</ol></div>
</div>
<div class="sect2" id="using_elements_correctly">
<div class="titlepage"><div><div><h3 class="title">Using Elements Correctly</h3></div></div></div>
<p id="the_element_bar">The Element Bar at the top of XXE will show you which element
        you have selected. <span class="emphasis"><em>Block elements</em></span> are
        paragraph-level elements (e.g., <code class="literal">programlisting</code>). <span class="emphasis"><em>Inline
        elements</em></span> go within block elements (e.g., <code class="literal">emphasis</code>). The Element Bar’s default
        description of text in any element is <code class="literal">#text</code>. You can select <code class="literal">#text</code> and convert to an element that better
        suits your needs, like <code class="literal">literal</code>.</p>
<p id="for_example_if">For example: if you want to emphasize a word or phrase by
        putting it in italics, highlight the text and click the Convert button
        in the Edit panel. In the field that appears, start typing the name of
        the inline element you wish to apply, in this case <code class="literal">emphasis</code>.</p>
<p id="the_convert_but">The Convert button is shown in <a class="xref" href="ch01.html#keyCapWrap" title="Figure 1-13. Using the Convert button">Figure 1-13</a>.
        You can also use the formatting buttons in the toolbar (above the
        Element Bar) to convert text to emphasis, links, plain text, etc. This
        is often faster than using the Convert button for these basic
        elements.</p>
<div class="figure-float"><div class="figure" id="keyCapWrap">
<div class="figure-contents"><div class="mediaobject"><img style="width: 378; " src="http://orm-chimera-prod.s3.amazonaws.com/1234000001577/images/keyCapWrap.png" alt="Using the Convert button"></div></div>
<div class="figure-title">Figure 1-13. Using the Convert button</div>
</div></div>
<p id="to_insert_a_bul">To insert a bulleted list, choose “Add itemizedlist” from the
        toolbar to insert the first bullet point. For a numbered list, choose
        “Add orderedlist.” For a term-definition list, choose “Add
        variablelist.”</p>
<p id="if_you_dont_se">If you don’t see the element you want as an option for your
        highlighted text, it probably means it isn’t a valid option. Try
        placing your cursor at a different level in the hierarchy. Only the
        elements that are valid for your location in the hierarchy according
        to the DTD are displayed as options. (This is a very useful feature of
        XXE as well as a potentially aggravating one, if you can’t figure out
        why the program is not letting you do something.)</p>
<p id="similarly_if_y">Similarly, if you’ve cut text from one spot and are having
        trouble pasting it, make sure you are pasting into the same element.
        XXE won’t let you paste somewhere invalid. Try moving up or down the
        hierarchy, or use the Paste, Paste Before..., or Paste After...
        buttons in the toolbar.</p>

<div class="sect2" id="inserting_hyperlinks">
<div class="titlepage"><div><div><h3 class="title">Inserting Hyperlinks</h3></div></div></div>
<p id="as_explained_in">As explained in <a class="ulink" href="http://chimera.labs.oreilly.com/books/1234000000058/ch02.html#inserting_hyperlinks" target="_top">“Hyperlinks”</a>
        in the DocBook guidelines, there are two types of hyperlinks you can
        create: those with character data and those without. Here’s the markup
        for each:</p>
<div class="variablelist" id="with_character_"><dl class="variablelist">
<dt><span class="term">With character data</span></dt>
<dd><p id="ulink_urlhtt_id1"><code class="literal">&lt;ulink
              url="http://oreilly.com"&gt;O'Reilly&lt;/ulink&gt;</code></p></dd>
<dt><span class="term">Without character data</span></dt>
<dd><p id="ulink_urlhtt_id2"><code class="literal">&lt;ulink
              url="http://www.oreilly.com"/&gt;</code></p></dd>
</dl></div>
<p id="to_add_a_hyperl_id1">To add a hyperlink with character data in XXE, type the
        character data as regular text (e.g., O’Reilly). Select the text,
        click the “Convert to link” button in the top toolbar, and choose
        <code class="literal">ulink</code> as shown in <a class="xref" href="ch01.html#ulink" title="Figure 1-21. Convert to ulink">Figure 1-21</a>.<a href="#ftn.id507480" class="footnote"><sup class="footnote" id="id507480">[1]</sup></a>If you have the ORM customizations installed, you will
        see a red message in the element that says, <span class="emphasis"><em>Empty
        &lt;ulink&gt;! Needs url&gt;&gt;</em></span>. Go to the Attributes
        panel and enter the <code class="literal">url</code> there (see
        <a class="xref" href="ch01.html#urlAttribute" title="Figure 1-22. Entering the URL">Figure 1-22</a>). It must be a full URL starting with
        <span class="emphasis"><em>http://</em></span>, not <span class="emphasis"><em>www</em></span>. This will
        produce a clickable link in the PDF and electronic versions of the
        book.</p>
<p id="to_add_a_hyperl_id2">To add a hyperlink without character data in XXE, simply insert
        a <code class="literal">ulink</code> element via the Edit panel, and fill in the
        <code class="literal">url</code> attribute as described above.</p>

<div class="sect2" id="working_with_index_terms">
<div class="titlepage"><div><div><h3 class="title">Working with Index Terms</h3></div></div></div>
<p id="the_index_is_au">The index is automatically generated from
        <code class="literal">indexterm</code>s in the component book files. See
        the <a class="ulink" href="http://atlas.labs.oreilly.com/books/1234000001578/" target="_top">O'Reilly
        Indexing Guidelines</a> for more information on indexing
        in DocBook. The following instructions assume the ORM customizations
        are installed.</p>
<div class="warning" id="if_you_are_an_a_id1"><p id="if_you_are_an_a_id2">If you are an author writing a first edition manuscript, you
          do <span class="emphasis"><em>not</em></span> need to worry about indexterms; indexing
          will be done by a professional indexer once your book is in
          production.</p></div>
<div class="sect3" id="inserting_a_new_index_entry">
<div class="titlepage"><div><div><h4 class="title">Inserting a new index entry</h4></div></div></div>
<p id="to_insert_a_new">To insert a new entry:</p>
<div class="orderedlist" id="place_the_curso_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="place_the_curso_id2">Place the cursor where you want the entry, and hit ESC-i.
              This will open a pop-up window that lets you type in primary,
              secondary, and tertiary index terms, separated by
              slashes.</p></li>
<li class="listitem"><p id="hit_enter_and_t">Hit Enter and the index entry will appear in gray.</p></li>
</ol></div>
<p id="to_insert_a_se">To insert a “See” or “See also” entry:</p>
<div class="orderedlist" id="after_creating__id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="after_creating__id2">After creating the entry as outlined above, select the
              <code class="literal">primary</code> element (or
              <code class="literal">secondary</code> or <code class="literal">tertiary</code>,
              depending on what the “See”/“See also” entry refers to) in the
              Element Bar to highlight it.</p></li>
<li class="listitem"><p id="select_insert_">Select “Insert After” (or hit Cmd-j) to insert the
              <code class="literal">see</code> or <code class="literal">seealso</code>
              element.</p></li>
<li class="listitem"><p id="enter_the_appro">Enter the appropriate text.</p></li>
</ol></div>
<p id="to_insert_an_in">To insert an index range:</p>
<div class="orderedlist" id="start_by_insert_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="start_by_insert_id2">Start by inserting a new entry as outlined above.</p></li>
<li class="listitem"><p id="select_the_inde">Select the <code class="literal">indexterm</code> in the Element Bar
              to highlight it. You should see a red box surrounding it.</p></li>
<li class="listitem"><p id="in_the_attribut_id1">In the Attributes panel, go to the “class” row and choose
              <code class="literal">startofrange</code>.</p></li>
<li class="listitem"><p id="with_indexterm_">With <code class="literal">indexterm</code> still highlighted, go to
              the “id” row and enter a name. The name can be anything you
              want, but we suggest you follow a format. For example, if you’re
              working in Chapter 2 and the term is <code class="literal">foo</code>,
              then the <code class="literal">id</code> might be
              <code class="literal">ch02_foo</code>.</p></li>
<li class="listitem"><p id="go_to_the_place">Go to the place in the text where the range should end,
              and insert another <code class="literal">indexterm</code> using ESC-i. But
              don’t actually enter a term in the pop-up window; just press
              “OK” to create an empty <code class="literal">indexterm</code>.</p></li>
<li class="listitem"><p id="select_the_empt">Select the empty <code class="literal">indexterm</code>.</p></li>
<li class="listitem"><p id="in_the_attribut_id2">In the Attributes panel, go to the “class” row and choose
              <code class="literal">endofrange</code>.</p></li>
<li class="listitem">
<p id="with_the_empty_">With the empty <code class="literal">indexterm</code> still
              highlighted, go to the “startref” row and enter the exact same
              <code class="literal">id</code> you used in step 4. If you don’t use the
              same text in the same case, you will get a validity error that
              looks something like this:</p>
<pre class="screen" id="reference_to_no">reference to non-existent ID "ch02_fot" [cvc-id.1]</pre>
</li>
</ol></div>
</div>
<div class="sect3" id="editing_existing_entries">
<div class="titlepage"><div><div><h4 class="title">Editing existing entries</h4></div></div></div>
<p id="to_search_for_i">To search for index entries using XXE:</p>
<div class="orderedlist" id="open_the_bookx_id1"><ol class="orderedlist" type="1">
<li class="listitem"><p id="open_the_bookx_id2">Open the <code class="filename">book.xml</code> file.</p></li>
<li class="listitem"><p id="with_the_cursor">With the cursor at the top of the document, hit ESC-G to
              open the Find Element pop up.</p></li>
<li class="listitem"><p id="in_the_element">In the “Element” field, type in
              <code class="literal">indexterm</code>. If you want to find a particular
              term, type it in the “Containing Text” field.</p></li>
<li class="listitem"><p id="hit_enter_to_ju">Hit Enter to jump to the first instance in the book. Hit
              Cmd-A to jump to the next instance.</p></li>
<li class="listitem"><p id="once_you_find_t">Once you find the entry you want, you won’t be able to
              edit it because the data is contained in the chapter file, not
              the <code class="filename">book.xml</code> file that you’re searching in.
              So hit Cmd-Shift-E to open the referenced file and jump to the
              entry.</p></li>
<li class="listitem"><p id="edit_and_save">Edit and save.</p></li>
<li class="listitem"><p id="go_back_to_book">Go back to <code class="filename">book.xml</code> to do your next
              search.</p></li>
</ol></div>
</div>
